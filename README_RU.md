# Прототип системы
Прототип должен уметь работать с файлом конфигурации и набором внешних событий определенного формата.
Решение должно содержать исходный файл/файлы на языке golang (1.22 или новее) и юнит-тесты (опционально).

## Описание
Игрок участвует в испытании. Цель — полностью зачистить подземелье. Игрок перемещается по этажам и сражается с монстрами. Нам нужно обработать события и собрать информацию в итоговый отчет.

## Правила
1.  К участию в испытании допускаются только зарегистрированные игроки.
2.  Испытание заканчивается, если:
    1.  Игрок покидает подземелье.
    2.  Игрок не может продолжать испытание.
    3.  Время открытия подземелья истекло.
    4.  Игрок мертв (здоровье упало до 0).
3.  При входе на этаж босса игрок получает уведомление.
4.  Этаж босса не содержит монстров.
5.  Подземелье считается пройденным, если:
    1.  Все этажи зачищены от монстров.
    2.  Босс побежден.
6.  Этаж считается пройденным, когда все монстры или босс убиты; ***любое время, проведенное на этом этаже после этого, больше не учитывается***.
7.  Здоровье игрока не может превышать 100.

## События
- Все события происходят последовательно во времени. (***Время события N+1***) >= (***Время события N***).
- Формат времени ***[ЧЧ:ММ:СС]***. Ведущие нули обязательны во входных и выходных данных.
- Параметр ***ExtraParam*** может быть строкой, содержащей несколько слов.

## Конфигурация (json)
- **Floors**        — Количество этажей в подземелье
- **Monsters**      — Количество монстров на каждом этаже подземелья
- **OpenAt**        — Время открытия подземелья
- **Duration**      — Время до закрытия подземелья в часах


#### Пример
```
{
    "Floors": 2,
    "Monsters": 2,
    "OpenAt": "14:05:00",
    "Duration": 2
}
```

## Статусы
| Статус    | Комментарий                                                           |
| ----------|:-----------------------------------------------------------------:|
|   SUCCESS |   Все этажи зачищены                                          |
|   FAIL	|	Игрок погиб или подземелье не считается пройденным      |
|   DISQUAL	|   Игрок не может продолжать или не прошел регистрацию    |

## Итоговый отчет
1.  Статус `SUCCESS`/`FAIL`/`DISQUAL`
2.  ID игрока
3.  Время, проведенное в подземелье (все время до момента, когда игрок покинул подземелье или оно закрылось)
4.  Среднее время зачистки этажа от монстров (этаж босса не включается в расчет)
5.  Время убийства босса
6.  Здоровье игрока в конце испытания

#### Пример
```
Final report:
[SUCCESS] 1 [00:24:00, 00:05:00, 00:11:00] HP:35
[FAIL] 2 [00:19:00, 00:00:00, 00:00:00] HP:0
[DISQUAL] 3 [00:00:00, 00:00:00, 00:00:00] HP:100
```
